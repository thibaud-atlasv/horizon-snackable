# UI Overlays

## Overview

All UI is built with XAML panels + `CustomUiComponent` + `UiViewModel`. Each UI element has:
- A `.xaml` file in `UI/` — layout and bindings
- A `UiViewModel` subclass — reactive data model
- A component — logic, animation, event subscriptions

UI and UiViewModel files are generated by the MHS assistant — do not create them manually.

## How to Add a New UI Overlay

1. **Define the ViewModel** in your component file:
   ```typescript
   @uiViewModel()
   export class MyOverlayData extends UiViewModel {
     titleText: string = '';
     isVisible: boolean = false;
   }
   ```

2. **Create the component**:
   ```typescript
   @component()
   export class MyOverlayComponent extends Component {
     private _vm = new MyOverlayData();

     @subscribe(OnEntityStartEvent)
     onStart(): void {
       if (NetworkingService.get().isServerContext()) return;
       const ui = this.entity.getComponent(CustomUiComponent);
       if (ui) ui.dataContext = this._vm;
     }
   }
   ```

3. **Request the XAML** from MHS assistant — describe the layout, bindings, and visual style

4. **Create a scene entity** in space.hstf with `CustomUiComponent` pointing to the XAML

5. **Subscribe to events** to show/hide and update the overlay

## Animation Pattern

All UI animations are driven from TypeScript by updating ViewModel properties each frame. No XAML storyboards.

```typescript
@subscribe(OnWorldUpdateEvent, { execution: ExecuteOn.Owner })
onUpdate(p: OnWorldUpdateEventPayload): void {
  if (!this._animating) return;
  this._animTime += p.deltaTime;
  const t = Math.min(this._animTime / DURATION, 1);
  this._vm.scaleX = easeOutElastic(t) * TARGET_SCALE;
  this._vm.opacity = t;
}
```

Common animation patterns used in this project:
- **Elastic pop-in**: Scale from 0 to target with overshoot (GameOverStats entrance)
- **Casino roll-up**: Count from 0 to N over time (ShotFeedback points)
- **Drain**: Element floats toward HUD position then disappears (points to score)
- **Explosion**: Scale up + fade out simultaneously (shot dots consumed)
- **Sine bob**: Looping vertical oscillation (instruction text)

## How to Modify the HUD

The HUD (`SoccerKickHudComponent`) polls `GameStateService.snapshot()` each frame and updates the ViewModel. To add a new HUD element:

1. Add a field to the ViewModel (e.g. `comboText: string = ''`)
2. Update the field in the frame loop based on snapshot data
3. Request XAML update from MHS assistant to add the visual element

## How to Modify the Game Over Screen

The Game Over screen (`GameOverStatsComponent`) listens to `PhaseChangedEvent` (GameOver) and runs an animation sequence. To add a new stat:

1. Add a ViewModel field (e.g. `newStatText: string = ''`)
2. Populate it from `GameStateService.snapshot()` when GameOver triggers
3. Add it to the animation sequence (count-up, delay, etc.)
4. Request XAML update from MHS assistant
