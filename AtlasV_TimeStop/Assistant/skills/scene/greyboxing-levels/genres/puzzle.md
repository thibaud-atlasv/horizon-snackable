# Puzzle Game Greyboxing

Priorities for puzzle game blockouts (Portal, The Witness, Baba Is You style).

## Teaching Without Words
- First encounter must allow experimentation
- Failure should be informative, not punishing
- Isolate new mechanics from previous complexity
- Environment teaches rules through consistent behavior

## Solution Clarity
- Intended solution should feel elegant when found
- Red herrings are acceptable; unfair tricks are not
- Multiple solutions can exist if all feel intentional
- Test if solution is reachable through logic, not guessing

## Visual Hierarchy
Even in greybox:
- Interactive elements distinct from scenery
- Puzzle inputs read differently from puzzle outputs
- State changes must be visible and clear
- Use consistent visual language across puzzles

## Puzzle Sequencing
Order puzzles by concept dependency:
1. **Foundation** — Core mechanic alone
2. **Extension** — Same mechanic, new context
3. **Combination** — Merge with previous concept
4. **Mastery** — Full complexity

## Aha Moment Design
The "aha" is the reward. Protect it:
- Don't over-hint; let players struggle productively
- Solution should feel earned, not given
- Test time-to-solve across skill levels
- Some players will be faster—that's okay

## Space for Thinking
Puzzle spaces need cognitive room:
- Minimize distractions in puzzle areas
- Player should be able to see all relevant elements
- Complex puzzles may need observation points
- Don't rush players with timers unless intentional

## Validation Method
Test with fresh players. Track:
- Time from encounter to solution
- Wrong approaches attempted
- Hint requests / give-up moments
- Post-solve satisfaction ("that was clever" vs. "finally")

## Checklist
- [ ] First encounter allows safe experimentation
- [ ] Solution reachable through logic
- [ ] Visual hierarchy clear in greybox
- [ ] Puzzle sequence builds concepts
- [ ] Aha moment protected
- [ ] Space allows cognitive processing
- [ ] Fresh player tested
